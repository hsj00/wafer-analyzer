
# 🔬 웨이퍼 맵 분석기 (Wafer Map Analyzer) v4.0

반도체 공정 데이터를 웹 브라우저에서 즉시 분석할 수 있는 Streamlit 기반 애플리케이션입니다.

CSV 또는 Excel 파일을 불러오거나 **데이터를 직접 입력**하여 **웨이퍼 맵 시각화 · 다중 웨이퍼 비교 · GPC 분석 · 결함 오버레이 · ML 이상 탐지 · Excel 보고서 생성**을 한 화면에서 처리합니다.

## 1. 이런 분께 필요합니다

* 박막 두께, 식각 깊이, 면저항 등 **웨이퍼 전면 분포 데이터**를 시각화하고 싶은 공정 엔지니어
* ALD 공정의 **GPC(Growth Per Cycle)** 균일도를 분석하고 싶은 연구자
* 결함 검사 장비 데이터를 공정 측정값과 **겹쳐서 보고 싶은** QC 담당자
* 여러 웨이퍼를 한꺼번에 비교하거나 **이상 웨이퍼를 자동으로 탐지**하고 싶은 엔지니어
* 데이터 파일 없이 **측정값을 직접 입력**해서 빠르게 맵을 확인하고 싶은 엔지니어

## 2. v4.0 주요 변경사항

### ① 한국어/영어 언어 선택 (신규)

사이드바 최상단에서 **한국어 / English**를 선택하면 앱 전체 UI가 해당 언어로 전환됩니다.

| 항목                | 설명                                       |
| ------------------- | ------------------------------------------ |
| 🌐 언어 선택 라디오  | 사이드바 최상단, 한국어/English 전환       |
| 📖 README 보기 버튼  | 선택된 언어에 맞는 README 전문을 메인 화면에 표시 |

### ② 파일·폴더 선택 — 팝업(Dialog) 방식 통일

기존 인라인 브라우저를 **@st.dialog** 기반 팝업으로 교체하여 Local/Cloud 환경에서 일관된 UX를 제공합니다.

| 환경  | 파일 선택                        | 폴더 선택                                |
| ----- | -------------------------------- | ---------------------------------------- |
| Local | @st.dialog 기반 인라인 브라우저  | 네이티브(tkinter) → 실패 시 dialog 폴백 |
| Cloud | @st.dialog 내 st.file_uploader   | N/A (파일 업로드 방식)                   |

### ③ Raw Data 편집 기능 강화

| 기능                  | 설명                                                    |
| --------------------- | ------------------------------------------------------- |
| 행 추가/삭제          | `num_rows="dynamic"` → 테이블 하단 +/- 버튼 활성화     |
| 편집 내용 유지        | `session_state["edited_df"]`에 저장 → 탭 이동 후에도 유지 |
| 편집 초기화 버튼      | 🔄 버튼으로 원본 데이터 즉시 복원                        |
| 차트 즉시 반영        | 편집된 DataFrame으로 차트·통계 자동 재계산               |
| 비교 모드 카드별 편집 | 각 비교 카드에서도 data_editor로 편집 가능               |

### ④ 비교 모드 데이터셋 이름 자동 입력

데이터셋 추가 시 **Data 컬럼 선택**이 변경되면 데이터셋 이름이 자동 갱신됩니다.
단, 사용자가 이름을 직접 수정한 경우에는 덮어쓰지 않습니다.

### ⑤ 성능·UX 개선

* 차트 생성 구간에 `st.spinner` 추가
* `apply_col_mapping` 컬럼 존재 확인 에러 핸들링 개선
* Cloud 환경 자동 감지 (`STREAMLIT_SHARING_MODE`, `/.dockerenv`)

## 3. 전체 기능 흐름

```
CSV / Excel 파일  또는  수동 입력(직접 타이핑 / 붙여넣기)
        │
        ▼
① 📊 웨이퍼 맵 ─┬─ 🔬 단일 분석  ← 파일 로드 또는 수동 입력으로 시작
                 └─ 🔀 비교 분석  ← 여러 웨이퍼/파라미터 나란히 비교
        │
        ├──► ② 🔍 결함 오버레이   별도 결함 CSV를 웨이퍼 맵 위에 겹쳐 표시
        │
        ├──► ③ ⚗️ GPC 분석        두께 ÷ 사이클 수, 반경별 균일도 프로파일
        │
        ├──► ④ 📄 보고서 생성     통계 + 차트 이미지 + 원본 데이터 → xlsx 다운로드
        │
        └──► ⑤ 🤖 ML 이상 탐지   여러 웨이퍼를 PCA + IsolationForest로 자동 분류
```

> **탭 ②~④**는 단일 분석에서 데이터를 먼저 로드한 후 사용하세요.
>
> **탭 ⑤**는 독립된 데이터셋 패널이 있어 탭 안에서 직접 파일을 추가할 수 있습니다.


## 5. 설치 및 실행

### 5-1. Python 버전 확인

Python **3.10 이상**이 필요합니다.

```bash
python --version
```

### 5-2. 가상환경 생성 (권장)

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 5-3. 패키지 설치

```bash
pip install -r requirements.txt
```

또는 직접 설치:

```bash
pip install streamlit pandas numpy plotly scipy openpyxl scikit-learn kaleido
```

| 패키지       | 용도                                | 없으면                  |
| ------------ | ----------------------------------- | ----------------------- |
| streamlit    | 웹 UI 프레임워크                    | 앱 실행 불가            |
| pandas       | CSV/Excel 읽기 및 데이터 처리       | 앱 실행 불가            |
| numpy        | 수치 연산                           | 앱 실행 불가            |
| plotly       | 인터랙티브 차트                     | 앱 실행 불가            |
| scipy        | 웨이퍼 격자 보간 (`griddata`)       | 앱 실행 불가            |
| openpyxl     | Excel 파일 읽기/쓰기                | 앱 실행 불가            |
| scikit-learn | ML 이상 탐지 (PCA, IsolationForest) | 🤖 ML 탭만 비활성화     |
| kaleido      | 차트 → PNG 변환 (보고서 이미지용)   | 이미지 없이 보고서 생성 |

### 5-4. 앱 실행

```bash
streamlit run app.py
```

브라우저에서 `http://localhost:8501`이 자동으로 열립니다.

### 5-5. 처음 실행 시

데이터 파일이 없어도 바로 시작할 수 있습니다. 두 가지 방법이 있습니다:

**방법 A — 샘플 데이터 생성:**

1. 앱 실행 후 왼쪽 사이드바에서 **"🎯 샘플 5개 생성"** 버튼을 클릭합니다.
2. `./wafer_data/` 폴더에 `wafer_01.csv` ~ `wafer_05.csv`가 자동 생성됩니다.

**방법 B — 수동 입력:**

1. 웨이퍼 맵 탭 → 단일 분석 서브탭에서 **"✏️ 수동 입력"**을 선택합니다.
2. 빈 테이블에 X, Y, Data 값을 직접 입력하거나 스프레드시트에서 붙여넣기(Ctrl+V)할 수 있습니다.
3. 유효 포인트 3개 이상이면 즉시 웨이퍼 맵이 생성됩니다.

## 6. 입력 데이터 형식

### 6-1. 기본 웨이퍼 맵 데이터 (CSV / Excel)

X·Y 좌표와 측정값 컬럼이 포함된 파일이면 됩니다.

| x     | y     | data  |
| ----- | ----- | ----- |
| 0.0   | 100.0 | 512.3 |
| -50.0 | 86.6  | 498.7 |
| -86.6 | 50.0  | 503.1 |

* **컬럼명 자동 인식:** `x`, `y`, `data` (대소문자 무관). 다른 이름이라도 사이드바 **🔗 컬럼 매핑**에서 직접 지정할 수 있습니다.
* **단위:** mm 기준. 200 mm 웨이퍼 → 반경 약 100 mm, 300 mm 웨이퍼 → 반경 약 150 mm
* **최소 포인트 수:** 보간 품질을 위해 10개 이상 권장 (수동 입력 시 최소 3개)

### 6-2. GPC 분석용 데이터

두께 컬럼과 사이클 수 컬럼이 추가로 필요합니다.

| x     | y     | thickness_nm | n_cycles |
| ----- | ----- | ------------ | -------- |
| 0.0   | 100.0 | 51.2         | 100      |

### 6-3. 결함 오버레이용 데이터 (별도 파일)

| x     | y     | class    | size | description   |
| ----- | ----- | -------- | ---- | ------------- |
| 10.5  | -20.3 | Particle | 5.0  | 큰 입자       |

* **결함 분류 컬럼:** `class`, `type`, `category`, `defecttype`, `label` 중 하나를 자동 인식합니다.

## 8. 탭별 사용 방법

### 탭 ① — 📊 웨이퍼 맵

#### 🔬 단일 분석 서브탭

**데이터 소스:** 📁 파일 데이터 또는 ✏️ 수동 입력

**제공 차트:** 2D Heatmap · Contour · Line Scan · 3D Surface

**Raw Data 편집 (v4.0 강화):**
* 행 추가/삭제: 테이블 하단 +/- 버튼
* 셀 편집: 직접 클릭하여 값 수정 → 차트 즉시 갱신
* 편집 초기화: 🔄 버튼으로 원본 복원
* 편집 내용은 탭 이동/rerun 후에도 유지됨

#### 🔀 비교 분석 서브탭

여러 웨이퍼 또는 같은 웨이퍼의 여러 파라미터를 나란히 비교합니다.

**v4.0 개선:** 데이터셋 추가 시 Data 컬럼 선택에 따라 이름이 자동 설정됩니다.

### 탭 ② — 🔍 결함 오버레이

웨이퍼 맵 위에 결함 좌표를 겹쳐 공간적 상관관계를 분석합니다.

### 탭 ③ — ⚗️ GPC 분석

ALD 공정의 GPC = 두께(nm) ÷ 사이클 수를 계산하고 공간 분포를 분석합니다.

### 탭 ④ — 📄 보고서 생성

분석 결과를 Excel(.xlsx) 파일로 내보냅니다.

### 탭 ⑤ — 🤖 ML 이상 탐지

PCA + Isolation Forest로 여러 웨이퍼 중 이상 패턴을 자동 탐지합니다.

## 9. 사용 예시

### 예시 A: 처음 실행 — 수동 입력으로 즉시 시작

```
1. streamlit run app.py
2. 웨이퍼 맵 탭 → 단일 분석 → ✏️ 수동 입력 선택
3. 빈 테이블에 좌표와 측정값을 직접 입력
```

```
1. 사이드바 최상단 🌐 Language에서 "English" 선택
2. 모든 UI가 영문으로 전환됨
3. 📖 View README 버튼으로 영문 README 확인 가능
```

```
3. 행 삭제: 불필요한 데이터 제거
```
